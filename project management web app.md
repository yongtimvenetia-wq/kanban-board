  
Goal:  
Website where I can manage projects using tickets, epics, and a Kanban board. The app should be clean, modern, responsive, and easy to use on desktop and mobile.

Core features:

Kanban board

* Columns: Backlog, To Do, In Progress, Review, Done  
* Ability to create, edit, delete, and move tickets between columns  
* Drag-and-drop ticket movement if possible  
* Tickets should display title, priority, status, assignee, due date, and epic if assigned

Tickets  
Each ticket should have:

* Ticket ID  
* Title  
* Description  
* Status  
* Priority: Low, Medium, High, Urgent  
* Assignee  
* Due date  
* Labels/tags  
* Optional epic  
* Created date  
* Updated date  
* Comments or activity log if possible  
3. Epics  
* Ability to create, edit, and delete epics  
* Epics should act as optional groupings/categories for tickets  
* I should be able to filter the board by epic  
* Each epic should have a name, description, color, and status

Meetings Notes tab  
For the first version, create a meetings page where I can:

* Create a meeting record  
* Add meeting title, date (automatically added), attendees (optional), and notes/transcript  
* Click a button to generate a structured meeting summary from the notes  
* The summary should include: key discussion points, decisions made, action items, owners, and follow-up items  
* For now, use a mock AI summary function or placeholder logic. Later I want to connect this to a real AI/transcription API.

General Notes tab

* An area where I can add notes (think of it as a scratchpad)  
* Ability to add special functions such as checkboxes and bulleted points  
* If possible, a way to filter, a way to then sort these notes by priority  
* If possible, add a way for me to add a priority label to a given note.
